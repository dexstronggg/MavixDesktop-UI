# MavixDesktop — Руководство оператора

Документ составлен в соответствии с ГОСТ 19.505-79 «Руководство
оператора. Требования к содержанию и оформлению».

---

## 1. Аннотация

Документ предназначен для оператора программы «MavixDesktop» — пилота,
осуществляющего управление БПЛА с персонального компьютера. В документе
изложены назначение, условия выполнения, последовательность работы и
сообщения, выдаваемые программой.

---

## 2. Назначение программы

«MavixDesktop» — программа для пилота, обеспечивающая дистанционное
управление дроном через интернет. Программа:

- отображает список собственных дронов;
- показывает живое видео с бортовых камер выбранного дрона;
- передаёт управляющие сигналы от джойстика на дрон;
- при работе с полётным контроллером MAVLink запускает программу
  QGroundControl с настройками текущего дрона.

---

## 3. Условия выполнения программы

### 3.1. Минимальный состав технических средств

- персональный компьютер с процессором архитектуры x86_64 или
  ARM64;
- объём оперативной памяти не менее 2 ГБ;
- видеокарта с поддержкой OpenGL 2.0;
- экран с разрешением от 1280×720 пикселей;
- джойстик с подключением по USB (необязательно — для управления);
- интернет-соединение со скоростью не менее 5 Мбит/с.

### 3.2. Минимальный состав программных средств

- операционная система: Ubuntu 22.04, Debian 12, Raspberry Pi OS
  Bookworm или совместимая;
- активное интернет-соединение;
- (опционально) программа QGroundControl, установленная и доступная в
  системном `PATH` или в стандартных каталогах размещения.

### 3.3. Подготовка к работе

Перед первым запуском пользователь должен иметь:

- действующую учётную запись в системе «Mavix» (адрес электронной
  почты и пароль);
- хотя бы один зарегистрированный дрон.

Регистрация выполняется через сервер (см. руководство оператора
сервера) или администратором системы.

---

## 4. Выполнение программы

### 4.1. Получение и установка

Установочный пакет `.deb` получается с сервера системы «Mavix» по
адресу:

```
GET https://<server>/api/v1/builds/desktop.deb
Authorization: Bearer <access_token>
```

Альтернативно — администратор системы предоставляет пакет напрямую.

Установка:

```bash
sudo apt install ./mavixdesktop.deb
```

В процессе установки автоматически скачиваются и устанавливаются все
необходимые Python-библиотеки. Длительность ~ 1–3 минуты в
зависимости от скорости интернет-соединения.

### 4.2. Первый запуск

Программа запускается из меню «Network → Mavix Desktop» или из
терминала:

```bash
/opt/mavixdesktop/.venv/bin/python -m mavixdesktop
```

При первом запуске отображается экран входа. Необходимо ввести
адрес электронной почты и пароль учётной записи. После успешного
входа открывается экран со списком дронов.

При повторных запусках вход выполняется автоматически — программа
использует ранее сохранённый refresh-токен из системного хранилища
секретов.

### 4.3. Выбор дрона и начало просмотра

На экране со списком отображаются карточки доступных дронов. Каждая
карточка показывает идентификатор дрона и его состояние
(«Готов» — дрон в сети, «Не в сети» — нет связи).

При нажатии на карточку готового дрона:

1. Программа отправляет запрос на установление соединения.
2. Открывается окно просмотра с панелью видео.
3. В течение нескольких секунд устанавливается WebRTC-сессия,
   и в окне отображается видео с бортовой камеры.
4. В нижней панели появляются настройки активной камеры (битрейт,
   разрешение) и индикатор полётного контроллера (если он подключён
   к дрону).

### 4.4. Переключение между камерами

При наличии нескольких камер переключение между ними выполняется
клавишами `←` и `→` либо соответствующими кнопками поверх видео.
Активной камере выделяется полная пропускная способность; остальным
автоматически снижается битрейт до 200 кбит/с.

### 4.5. Изменение параметров камеры

В нижней панели окна просмотра доступны:

- выпадающий список параметров (разрешение и частота кадров);
- ползунок битрейта;
- кнопка «Сохранить» — отправка изменений на дрон.

После нажатия «Сохранить» дрон применяет новые параметры в течение
одной секунды.

### 4.6. Запуск спид-теста

В нижней панели расположена кнопка «▶ Speed test». При её нажатии
запускаются три параллельных теста:

- скорость отдачи (`Download`);
- скорость передачи (`Upload`);
- задержка до сервера и до дрона.

Результаты отображаются накладкой поверх видео. Для остановки —
нажать ту же кнопку ещё раз.

### 4.7. Подготовка к полёту

1. Подключить джойстик к компьютеру.
2. Нажать «Управление» на экране просмотра.
3. Выполнить процедуру калибровки джойстика по подсказкам на экране
   (центр, максимумы по каждому каналу, кнопка ARM). Результат
   сохраняется в `~/.config/mavixdesktop/data/<имя джойстика>.json`.
4. При повторном использовании того же джойстика калибровка
   подгружается автоматически.

#### 4.7.1. Раскладка осей пульта Radiomaster Pocket (Mode 2, EdgeTX)

Программа использует стандартную раскладку TAER1234 (Throttle —
Aileron — Elevator — Rudder).

| Ось pygame | Стик пульта | Канал CRSF | Функция |
|---|---|---|---|
| A0 | Правый, ось X | CH2 | Крен (Roll) |
| A1 | Правый, ось Y | CH3 | Тангаж (Pitch) |
| A2 | Левый, ось Y | CH1 | Тяга (Throttle) |
| A3 | Левый, ось X | CH4 | Рыскание (Yaw) |
| A4 и выше | Тумблеры верхней панели | CH5 и выше | Тумблер ARM и др. |

> На пульте Radiomaster Pocket тумблеры экспортируются как **оси**, а
> не как кнопки. Программа автоматически определяет тип при
> калибровке.

#### 4.7.2. SDL-строка для QGroundControl

После калибровки джойстика на экране калибровки доступна кнопка
«Копировать SDL-строку». Полученное значение записывается в переменную
окружения `SDL_GAMECONTROLLERCONFIG` при автоматическом запуске
QGroundControl. Это позволяет QGroundControl правильно интерпретировать
оси и кнопки джойстика. При ручном запуске QGroundControl значение
можно установить вручную:

```bash
export SDL_GAMECONTROLLERCONFIG="<скопированная строка>"
QGroundControl
```

### 4.8. Порядок включения дрона перед началом полёта

> Перед всеми проверочными тестами в помещении обязательно снять
> пропеллеры.

1. Открыть в наземной станции окно полёта.
2. Убедиться, что в журнале наблюдается активное видео и доступен
   полётный контроллер (индикатор внизу экрана).
3. Установить стик газа в крайнее нижнее положение.
4. Отсоединить кабель USB от полётного контроллера дрона (если был
   подключён для настройки) — арм при подключённом USB запрещён в
   большинстве прошивок.
5. Подключить аккумулятор к дрону.
6. Подождать 5 секунд (полётный контроллер инициализируется, бортовое
   программное обеспечение его обнаруживает).
7. Перевести тумблер «ARM» на джойстике в положение «включено».

> Если при входе в полётный режим тумблер «ARM» уже стоит в положении
> «включено», приложение остаётся в DISARM (не армирует автоматически).
> Это защита от случайного арма сразу при входе. Переведите тумблер в
> «выключено» и снова в «включено» — после этого арм сработает.

Признаком успешного арма является изменение индикации на полётном
контроллере (для большинства прошивок — переход с красного на
синий) и тихий характерный звук двигателей.

### 4.8. Полётный режим

После выбора джойстика:

- если на дроне работает MAVLink — программа автоматически
  запускает QGroundControl (при наличии в системе) и проксирует
  через себя UDP-канал между QGC и дроном; QGroundControl
  принимает все полётные команды. Пока идёт поиск QGroundControl,
  показывается окошко «Ищу QGroundControl…» (не дольше нескольких
  секунд) — программа при этом не зависает. Если найти исполняемый
  файл не удалось, программа предложит указать его вручную (`.exe`
  в Windows или `.AppImage`/бинарь в Linux); выбранный путь
  запоминается и используется при следующих запусках. Если позже
  файл по сохранённому пути исчезнет, программа снова попросит
  указать путь;
- если на дроне работает CRSF — открывается полноэкранное окно с
  отображением видео и состояния стиков. Управляющие пакеты CRSF
  формируются программой и отправляются на дрон в режиме реального
  времени.

Выход из полётного режима — клавиша `Esc` или закрытие полноэкранного
окна.

### 4.9. Возврат к списку дронов

Кнопка «Назад» на экране просмотра отключает сессию и возвращает
пользователя к списку. После этого можно выбрать другой дрон или
выйти из программы.

### 4.10. Завершение работы

Закрытие главного окна.

### 4.11. Выход из учётной записи

Кнопка «Сменить учётную запись» (на экране списка дронов) удаляет
refresh-токен из хранилища секретов и возвращает программу на экран
входа.

## 5. Сообщения оператору

### 5.1. Сообщения в окне программы

| Сообщение | Значение |
|---|---|
| `Email and password are required` | На экране входа не заполнены поля |
| `Invalid credentials` | Неправильный логин или пароль |
| `streaming...` | Видеосессия устанавливается |
| `no video` | Сессия не установлена; видео отсутствует |
| `QGroundControl already running` | Открыто другое окно QGC; необходимо закрыть его |

### 5.2. Возможные нештатные ситуации

| Симптом | Вероятная причина | Действия |
|---|---|---|
| Не открывается экран входа | Программа установлена не полностью | Проверить установку: `dpkg -s mavixdesktop` |
| Не удаётся войти | Нет интернета или недоступен сервер | Проверить интернет, состояние сервера у администратора |
| Список дронов пуст | Не зарегистрирован ни один дрон | Зарегистрировать дрон через систему «Mavix» |
| Дрон в списке, но «Не в сети» | Дрон выключен или потерял связь | Включить дрон, проверить интернет на дроне |
| Видео не появляется | Не открылись data-каналы или WebRTC | Подождать ~10 секунд; проверить, что у дрона есть STUN/TURN сервер |
| Видео идёт с задержкой / пропадает | Узкий канал связи | Уменьшить битрейт в нижней панели; запустить спид-тест для диагностики |
| Джойстик не определяется | Не подключён или нет драйверов | Проверить подключение, выполнить `ls /dev/input/js*` |
| QGroundControl не запускается | Не установлен или не найден автоматически | Указать путь к исполняемому файлу в появившемся диалоге, либо установить QGroundControl и перезапустить программу |
| Снова просит указать путь к QGroundControl | Ранее выбранный файл перемещён или удалён | Указать актуальный путь к исполняемому файлу QGroundControl |

### 5.3. Журнал работы

При обращении в техническую поддержку приложить файл журнала:

```
~/.config/mavixdesktop/logs/mavixdesktop_<дата>.log
```

## 6. Углублённая диагностика связи

### 6.1. Видео не появляется в окне просмотра

Возможные причины расположены в порядке убывания вероятности:

1. **WebRTC-сессия не успела установиться.** Соединение требует
   ~5–10 секунд (обмен SDP, ICE, DTLS-рукопожатие). Подождать.
2. **Недоступен STUN-сервер.** В случае работы через NAT или
   симметричный файрвол ICE-обмен может не завершиться без рабочего
   STUN/TURN. Проверить, что сервер `Mavix` отдаёт корректные адреса
   по адресу `/api/v1/ice-servers`.
3. **Заблокирован UDP в сети.** Корпоративные сети нередко режут
   исходящий UDP-трафик. WebRTC по умолчанию использует UDP. В этом
   случае требуется TURN-сервер с поддержкой TURN-over-TCP.
4. **На дроне не запущены камеры.** Проверить журнал дрона командой
   `journalctl -u mavixboard -n 200` на стороне дрона.

### 6.2. Видео с задержкой или прерывистое

| Признак | Действие |
|---|---|
| Заметная задержка (> 0,5 сек), плавное видео | Запустить спид-тест; при низкой скорости отдачи дрона уменьшить битрейт активной камеры |
| Прерывистое видео, частые «фризы» | Низкое качество канала связи дрона; сменить точку доступа / оператора LTE |
| Изображение зеленеет / артефакты | Перегрузка кодека на стороне дрона: уменьшить разрешение или битрейт |

### 6.3. WebRTC: общие диагностические сообщения

Сообщения наблюдаются в файле журнала `~/.config/mavixdesktop/logs/...`.

| Сообщение | Причина | Действие |
|---|---|---|
| `[coord] signal closed: ConnectionClosedError 1006` | Внеплановый разрыв WebSocket | Программа автоматически переподключается; проверить интернет |
| `[coord] auth_warning: 60 seconds left, refreshing` | Скоро истекает access-токен | Штатная ситуация; обновление происходит автоматически |
| `[coord] auto-reconnecting to drone <id>` | Дрон вернулся в сеть после отключения | Штатная ситуация |
| `Could not bind to UDP port 14550` | QGroundControl уже занял порт | Закрыть QGroundControl, перезапустить «Mavix Desktop» |

### 6.4. Полётный контроллер не армируется

После получения сообщения «не армируется» (нет смены индикации или
двигатели не дают звукового сигнала):

1. Открыть журнал дрона (через `journalctl -u mavixboard -n 200`
   на бортовом компьютере или через систему мониторинга).
2. Убедиться, что в журнале присутствует строка
   `[fc-service] FC connected: <тип> / <имя>` — иначе бортовое ПО
   не нашло полётный контроллер.
3. При наличии подключения проверить флаги в конфигураторе
   Betaflight/iNav (вкладка CLI, команда `status`) или в QGroundControl
   (для ArduPilot/PX4).

Подробный перечень причин и действий по каждой прошивке приведён в
руководстве оператора программы «MavixBoard», раздел «Устранение
неполадок».

### 6.5. Сброс настроек джойстика

В случае некорректной работы джойстика после калибровки удалить файл
конфигурации:

```bash
rm ~/.config/mavixdesktop/data/<имя_джойстика>.json
```

После следующего запуска программа предложит выполнить калибровку
заново.

### 6.6. Сброс сохранённых учётных данных

Команды (при наличии gnome-keyring):

```bash
secret-tool clear service mavixdesktop
```

Альтернативно — удалить резервный файл:

```bash
rm ~/.config/mavixdesktop/tokens.json
```

После следующего запуска программа покажет экран входа.
